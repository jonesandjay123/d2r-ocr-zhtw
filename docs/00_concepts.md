# OCR 訓練核心概念（人話版）

## 4 個你需要懂的概念

### 1. GT（Ground Truth）= 標準答案
每張訓練圖都有一個 `.gt.txt` 檔，裡面寫這張圖的「正確答案」。
例如：一張寫著「闊劍」的圖，對應的 gt.txt 就寫 `闊劍`。

### 2. 字集（Charset / Unicharset）= 模型允許輸出的字
就像考試的選擇題範圍。如果「闊」不在字集裡，模型永遠輸出不了「闊劍」。
我們的字集 = D2R 所有物品名用到的中文字（~1245 個）。

### 3. Overfit Test = 用小資料確認流程沒壞
只用 200 張圖訓練，看模型能不能讀對。
能 → 流程 OK，可以擴大。不能 → 流程有 bug，先修。
就像先試做 1 件衣服確認版型對，再大量生產。

### 4. Eval on Real Crops = 真實遊戲截圖才是期末考
合成圖再像，也不是真的遊戲畫面。最終要用真實 crop 測試才算數。

## 常見的「儀表板數字」

| 數字 | 意思 | 好的值 |
|------|------|--------|
| BCER | 字元錯誤率 | 越低越好（<5% 算好） |
| iterations | 訓練跑了幾輪 | 不是越多越好，看 BCER 是否還在降 |
| skip ratio | 被跳過的樣本比例 | <5% 正常，>10% 代表有問題 |
